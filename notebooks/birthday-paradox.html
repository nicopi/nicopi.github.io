The Birthday Paradox¶

The birthday paradox is a well known problem in probability theory. The question is the following:

given a set of $n$ randomly chosen people, which is the probability that at least two of them have the same birthday?

The probability reaches 100% when the number of people reaches 367 (since there are only 366 possible birthdays, including February 29). However, it can be demonstrated that 99.9% probability is reached with just 70 people, and that a 50% probability can be reached with 23 people.

To simplify the problem we don't consider leap years, twins, seasonal, or weekday variations, and we assume that all 365 possible birthdays are equally likely (that's not true in reality, however this will not affect much our results).

In the following, we consider $E$ the event "at least two people have the same birthday".

Numerical solution¶

In this first part, we attempt finding a solution to our question by performing a numerical simulation.

We define the function test(n) that returns the number of people having the same birthday as someone else in a group of $n$ people.

In [1]:
import numpy as np
import matplotlib.pyplot as plt
%matplotlib inline
import random
import scipy.special as sp
import math

#return the number of people with the same birthday
def test(n):
    days=np.zeros([365,1])
    for i in range(n):
        random_day = random.randrange(0, 365) #extract a random day in [1,365]
        days[random_day,0]+=1
    return np.count_nonzero(days>1)

As an example, let's see how many people have the same birthday as someone else in a group of 50 people.

In [2]:
test(50)
Out[2]:
1

The result will change every time we call the function since the birthdays are decided randomly.

We define the function trials(n,ntrials) that allows performing multiple tests considering a group of $n$ people. The variable ptrials counts the number of positive trials, i.e., the number of trials in which there are at least two people with the same birthday. The function returns the probability \begin{equation} P(E) = \frac{\text{ptrials}}{\text{ntrials}} \end{equation}

In [3]:
def trials(n,ntrials):
    ptrials=0
    for i in range(ntrials):
        out=test(n)
        if out>0:
            ptrials+=1
    return ptrials*1.0/ntrials

Let's compute the probability of the event $E$ when considering 50 people based on the results of 1000 trials.

In [4]:
trials(50,1000)
Out[4]:
0.964

Finally, let's compute the probability of the event $E$ for different values of $n$.

In [5]:
probabilities = np.zeros([100,1])
for i in range(1,100):
    probabilities[i,0]=trials(i,1000)
plt.plot(np.r_[1:101], probabilities);
plt.xlabel('$n$');
plt.ylabel('$P(E)$');
plt.grid()

It may be surprising that a group of 23 people is sufficient to have a 50% probability of having two people with the same birthday. Moreover, 99.9% probability is reached with just 70 people.

Analytical solution¶

In this section, we analytically find the probability of the event $E$.

The probability of $E$ can be written as \begin{equation} P(E) = 1 - P(\bar{E}) \end{equation} where $\bar{E}$ is the event: "no two people in the room have the same birthday".

This step allows simplifying the calculations. Let's consider, for instance, a group of $n=3$ people. The probability of the event $\bar{E}$ can be computed as: \begin{equation} P(\bar{E}) = \frac{365}{365} \cdot \frac{364}{365} \cdot \frac{363}{365} = \left(\frac{1}{365}\right)^3 \cdot 365 \cdot 364 \cdot 363 \end{equation} The rationale behind this formula is pretty easy. The first birthday can be any of the 365 days (i.e., $\frac{365}{365}$), the second cannot be the same as the first (i.e., $\frac{364}{365}$), the third cannot be the same as the first two (i.e., $\frac{364}{365}$).

The obtained equation can be generalized for $n$ people: \begin{equation} P(\bar{E}) = \frac{365!}{365^n(365-n)!} \end{equation} The main obstacle for the computation of this probability is represented by the factorial that will return an overflow error. However, reminding the definition of the binomial coefficient (i.e., ${N\choose k}=\frac{N!}{k!(N-k)!}$), we can write \begin{equation} P(\bar{E}) = \frac{n! {365 \choose n}}{365^n} \end{equation} This function can easily be implemented by adopting the binom function included in the scipy library.

We define the function prob(n) that returns the probability of $E$ for a group of $n$ people:

In [6]:
def prob(n):
    return 1-math.factorial(n)*sp.binom(365,n)/(365**n)
prob = np.vectorize(prob)

Let's compute (again) the probability of the event $E$ when $n=50$.

In [7]:
prob(50)
Out[7]:
array(0.97037358)

This confirms that the result obtained through simulation was very close to the exact one!

Finally let's compute again $P(E)$ for different sizes of the group.
